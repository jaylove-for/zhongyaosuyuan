# 管理者后端管理平台 (Admin Dashboard) 需求规格说明书

## 1. 产品简介
本项目为“乡土印迹”系统的独立化管理后台，旨在将原前后端耦合在同一站点的管理权限抽离，形成专门供管理员和运营人员使用的工作台。

## 2. 核心架构说明
- **前端架构**: 采用 React 19 + Vite + Tailwind CSS 4。
- **UI 组件**: 采用响应式设计，搭配 Lucide React 提供丰富的图标库。
- **部署位置**: 独立运行与部署，通过鉴权接口确保安全隔离。

## 3. 功能模块规划

### 3.1 权限与登录模块
- **管理员登录**: 独立的管理员登录界面。
- **会话管理**: 基于 Token 或 Cookie 持久化登录状态。严防未授权的越权访问。

### 3.2 控制台总览 (Dashboard)
- **核心数据看板**: 汇总展示平台用户总数、上链信息统计、交易流水等宏观指标。
- **系统状态**: 后端服务运行监控、API 调用量等。

### 3.3 业务管理模块
- **用户管理**: 查看注册用户列表，禁用/解禁违规用户。
- **溯源数据管理**: 审核前端提交的乡土产品溯源信息。
- **GIS 与模型管理**: 对接管理 GIS 驾驶舱底图参数及 VR 博物馆的内容上下架。

## 4. 安全与非功能性需求
- **接口安全**: 操作涉及敏感数据时拦截检查 Admin 权限。
- **稳定性**: 所有的 API 请求需进行 Try-Catch 错误处理，前端应有完整的 fallback 状态和 ErrorBoundary。
